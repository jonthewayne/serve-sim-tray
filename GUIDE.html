ServeSim Tray

Run an iOS Simulator as a live, controllable stream — locally, or shared over Tailscale.

First-time setup — paste this to your coding agent

Set up this Mac to run ServeSimTray (an iOS-simulator streaming menu-bar app). Do each step and
pause to ask me when you need input (App Store sign-in, Tailscale login, enabling Serve):

1. Ensure Xcode is installed (Mac App Store) and an iOS runtime is present:
   sudo xcode-select -s /Applications/Xcode.app && xcodebuild -downloadPlatform iOS
2. Ensure Node 20+ is installed (e.g. via nvm).
3. (Optional, for remote/other-device access) Ensure Tailscale is installed and signed in, and
   enable Tailscale Serve for my tailnet at login.tailscale.com. Skip this for local-only use.
4. In the serve-sim-tray folder run ./build.sh install (compiles locally + copies to /Applications —
   no code signing needed), then launch it and turn on "Start at Login" from its menu.
Then open the app's "Check Setup…" to confirm everything is green.

No agent? See the full walkthrough in README.md. Or just click Check Setup… in the menu.

Two ways to use it

  • Local: just hit Start and open http://localhost:3200 — or point a local AI agent at it. No Tailscale needed.
  • Over Tailscale: with Tailscale up + Serve enabled, the same stream is reachable from your phone, other computers, or a remote agent at your tailnet URL (Copy Sim URL).
  • Public (Settings ▸ Share Publicly): Tailscale Funnel exposes the stream to the whole internet — a friend just opens the link. Anyone with the link can view and control the sim (no password), so only share it with people you trust.

Remote control — start it from another machine (no SSH)

The app runs a tiny HTTP endpoint on this Mac's Tailscale IP, port 8765 — reachable only by your tailnet devices, from anywhere (no shared Wi-Fi needed). Three scoped actions, no shell: /status · /start · /stop. Grab the base URL from Settings ▸ Copy Remote-Control URL.

# From another tailnet machine (e.g. your dev box). MAIN = this Mac's name or its 100.x IP.
MAIN=curiouss-macbook-air

# Available? (host must be awake + logged in + running the app)
curl -s -m 3 "http://$MAIN:8765/status"   # {"running":..,"url":".."}  — or times out if it's off

# Start serving, wait until ready, then use the url it returns:
curl -s "http://$MAIN:8765/start"
until curl -s -m 3 "http://$MAIN:8765/status" | grep -q '"running":true'; do sleep 2; done

# Open the "url" from /status in a browser — it's not just a picture: clicks, typing,
# swipes, and drag-dropped files in that page are FORWARDED TO THE SIMULATOR
# (MJPEG stream out + WebSocket control channel back in). A remote human — or a
# browser-driving agent — can fully drive the sim this way over Tailscale.
#
# Agents: the Tools panel's AX TREE toggle exposes the sim's accessibility tree in
# the page DOM (element names + bounds) — read it for semantic targets. But the
# overlay INTERCEPTS clicks while on: read with it ON, then toggle OFF to click.

# Stop when done:
curl -s "http://$MAIN:8765/stop"

If /status times out, the host is off or logged out — there's no remote wake (by design). Note: serve-sim's bundled Agent Skill (CLI taps/gestures) only works for agents running on this Mac — remote agents drive via the browser as above.
